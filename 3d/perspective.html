transform:perspective vs perspective

  • The perspective property defines how far the object is away from the user. So, a lower value will result in a more intensive 3D effect than a higher value.
  • When defining the perspective property for an element, it is the CHILD elements that get the perspective view, NOT the element itself.
  • Using perspective (with a value different from 0 or none) creates a new stacking context.
  • Default perspective is none and it's the same for 0
  • perspective set in parent carries to its children. Which is equivalent to set transform:perspective(50em) rotateX(50deg); for all children
  • Default perspective-origin is 50% 50%
